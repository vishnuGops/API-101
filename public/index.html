📖 Quick Reference

🟢 GET

Retrieves data from the server. Should not modify any data. Example: GET /api/books

🔵 POST

Creates new resources on the server. Sends data in the request body. Example: POST /api/books

🟠 PUT

Replaces an entire resource. Requires ALL fields. Example: PUT /api/books/1

🟣 PATCH

Partially updates a resource. Only send fields you want to change. Example: PATCH /api/books/1

🔴 DELETE

Removes a resource from the server. Example: DELETE /api/books/1

⚪ HEAD

Like GET but returns only headers, no body. Great for checking if a resource exists.

📍 Path Parameters

Dynamic parts of the URL that identify resources. Example: /api/books/:id where :id is 1, 2, etc.

❓ Query Parameters

Key-value pairs after ? for filtering. Example: /api/books?genre=Tech&limit=5

📋 Headers

Metadata about the request like Content-Type, Authorization, and custom headers.

📄 Pagination

Split large datasets into pages. Use ?page=2&limit=10 to navigate through results.

🚦 Rate Limiting

APIs limit requests per time window. Check X-RateLimit-Remaining header. 429 = too many requests.

🔑 Idempotency

Same request = same result. Use X-Idempotency-Key header to prevent duplicate operations.

🔗 Nested Resources

Resources belonging to others: /books/1/reviews shows reviews for book 1.

🏷️ API Versioning

Evolve APIs without breaking clients: /api/v1/ vs /api/v2/ for backward compatibility.

⏰ ETags & Caching

ETags identify resource versions. Send If-None-Match to get 304 (use cache) if unchanged.

🔔 Webhooks

Server pushes events to YOUR URL when things happen. Reverse of normal API calls.
